Backprop, Animated

Forward fills the data (left → right). Backward fills the grad (right → left). Press the buttons and watch every box light up in order — first one weight, then a whole little network.

1Reminder: every box has two slots

Each circle/box in the animations below is a Value — and it holds two numbers: data (its value) and grad (its effect on the error). Forward fills the first slot; backward fills the second.

2One weight: the trail w → d → e

This is the (w−10)² example, broken into its two real steps: d = w − 10, then e = d² (the error). Watch data fill going right, then grad fill coming back.

forward — filling data backward — filling grad

3Now scale up: a 13-weight network

Exactly the same two phases — just more boxes. This little network has 2 inputs → 3 hidden → 1 output = 13 parameters (9 weights + 4 biases). Watch the forward wave fill all the data, then the backward wave push grad back through every weight.

forward wave (data) backward wave (grad) not yet reached

4The takeaway

Both animations are the same machine:

  • FORWARD — run left → right, filling every box's data (this computes the error).
  • BACKWARD — start with grad = 1 at the error, run right → left, multiplying local rates to fill every box's grad.
  • Then nudge every weight: new = old − step × grad, and repeat.

One weight or 13 weights or 175 billion weights — identical process. The only thing that grows is the number of boxes. That's backpropagation.
