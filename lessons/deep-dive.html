1Parameters vs Weights

Short answer: weights and biases are the two kinds of parameter. "Parameters" is the umbrella word for all the learnable numbers; weights are the big group, biases are the small group.

TermWhat it isLives on…
WeightA number on a connection between two neurons — "how much does this input matter?"every arrow/line
BiasA single number added inside a neuron — shifts its output up/downevery neuron (one each)
ParameterThe umbrella term: every weight + every bias = all the learnable numbersthe whole model

In casual speech people say "weights" to mean all parameters (e.g. "the model's weights"). Strictly: parameters = weights + biases. When you hear "7 billion parameters," that's every weight and bias counted together.

Count them yourself, with real numbers

Here's a tiny network: 2 inputs → 3 hidden neurons → 1 output. Every line is a weight; every neuron in the hidden/output layers has a bias. The numbers shown are made-up "trained" values.

Purple/green numbers = weights (one per line). Yellow = biases (one per neuron).
Counting the parameters in this tiny net:
  weights input→hidden: 2 × 3 = 6
  weights hidden→output: 3 × 1 = 3
  biases (h₁,h₂,h₃,y): 4
  ──────────────
  total parameters = 6 + 3 + 4 = 13
Scale it up: GPT-3 does this exact counting across ~96 layers of thousands of neurons each → 175,000,000,000 numbers. Same idea, just astronomically more lines and neurons. Each of those 175B numbers was set by the process in Section 2 ↓

2Gradient vs Gradient Descent

A gradient is a number (a slope). Gradient descent is the action of using that number to step downhill. One is the compass; the other is the walking.

  • Gradient = "if I nudge this weight up a little, how much does the error change, and in which direction?" Its sign tells you which way is uphill; its size tells you how steep. For a whole model with millions of weights, the gradient is one slope per weight — a long list of numbers (yes, a vector — ties into Section 3).
  • Gradient descent = the update rule applied over and over:
    new weight = old weight − (learning rate × gradient)
    The minus sign is the whole trick: gradient points uphill, so we step the opposite way, downhill, toward less error.

Watch it with real numbers

Say the error depends on one weight w by the bowl-shaped curve loss = (w − 3)². The lowest error is obviously at w = 3. For this curve the gradient (slope) is 2 × (w − 3). Start at w = 8 and let gradient descent find the bottom — every column is a real computed number.

stepw (weight)loss = (w−3)²gradient = 2(w−3)next w = w − lr·grad

Watch the loss column shrink toward 0 and w march toward 3. Notice the gradient shrinks as you near the bottom (the curve flattens), so the steps get gentler automatically. Try the 1.05 learning rate to see steps overshoot and the loss explode — that's why learning rate matters.

The link to Part 1's rolling ball: the ball's position = w; the hill's steepness under it = the gradient; each roll-step = one line of this table. Now you've seen the same thing as a picture and as numbers.

3Vectors & Embeddings — in numbers and in 3D space ★

A vector is nothing scary: just an ordered list of numbers. An embedding is the vector that represents a word's meaning. "In space" just means: treat those numbers as coordinates and plot the point.

First, "in numbers"

A 3-number (3-dimensional) vector [x, y, z] is a point in 3D space — same as a pin on a map but with a third axis. Here are toy 3-D embeddings for a few words:

cat = [0.80, 0.60, 0.20] dog = [0.90, 0.50, 0.30] king = [-0.70, 0.80, 0.30] apple = [0.10, -0.70, 0.60]

Real embeddings aren't 3 numbers — they're hundreds or thousands (e.g. [0.02, -0.41, …, 0.77] with 1536 entries). We can't see 1536 dimensions, so this page uses 3 so it fits in space you can actually look at. The idea is identical; there are just more numbers.

Now, "in space" — drag to rotate

Below, each word's 3 numbers are plotted as a point in 3D. Drag to spin it. Notice meaning becomes geography: animals huddle together, royalty sits apart, food drops below. Closeness = similar meaning.

Measure "closeness" yourself

Pick two words. We compute the straight-line distance between their points (smaller = more similar in meaning). This exact "find the nearest vectors" operation is what a vector database does in RAG.

Why this is the whole point: once meaning is numbers in space, the computer can do math on meaning — measure similarity (distance), find related items (nearest points), even do analogies (king − man + woman ≈ queen is literally vector subtraction/addition). That's the engine under search, recommendations, RAG, and an LLM's understanding of words.
