1The problem neural networks solve

Normal programs follow rules you write. A neural network is different: you never write the rule. You show it examples, and it figures out the rule by itself.

🖊️ Old way: you write rules

You sit down and try to define "what makes a photo a cat." Pointy ears? Whiskers? Fur? You'd write thousands of rules and still fail — a cat from behind has no whiskers in view.

🧠 New way: it learns rules

You show it 50,000 photos labeled "cat" or "not cat." It discovers the pattern on its own — including patterns no human could write down. That discovered pattern lives inside the network as numbers.

The single most important sentence on this page: A neural network is just a big pile of numbers (called weights) plus a simple formula that turns an input into an output. "Learning" = slowly adjusting those numbers until the answers get good.

2The neuron — the tiny building block

Everything is built from one humble unit: the neuron. A single neuron does just three steps. That's the entire thing. Look at the diagram, then we'll read it left to right.

A single neuron: inputs come in, get weighted & summed, a bias is added, then squished through an activation.

Reading it in plain English

  • Step 1 — Weight & sum. Each input x is multiplied by its own weight w (a number saying "how much does this input matter"), and the results are added together.
  • Step 2 — Add the bias. A single extra number, the bias b, is added. It shifts the result up or down — think of it as the neuron's "default mood" before any input arrives.
  • Step 3 — Activation. The total is passed through an activation function f() that decides the final output. (Why we need this is Section 4 — it's the secret sauce.)
The whole neuron in one line:
output = f( x₁·w₁ + x₂·w₂ + b )
The weights (w) and the bias (b) are the only things that get learned. Inputs come from your data. That's it — memorize this and you understand 90% of neural networks.

4The activation function — why it exists

Here's a subtle but crucial idea. If a neuron only did "multiply and add," then stacking a million neurons would still collapse into one big straight-line equation. It could only learn straight-line relationships — useless for real problems like images or language, which are full of curves and corners.

The activation function bends the line. It injects non-linearity, and that bend is what lets networks learn complex shapes. Here are the three you'll hear about constantly:

The three shapes. Each "bends" the straight line in a different way.
NameWhat it doesWhen it's used
ReLUIf the number is negative, output 0. Otherwise keep it. Dead simple, very fast.The default inside almost every modern network.
SigmoidSquishes any number into a value between 0 and 1.Output layer when you want a probability ("80% cat").
SoftmaxTurns a list of numbers into probabilities that add up to 100%.Output layer when choosing among many options (e.g. the next word).

You saw ReLU, sigmoid and tanh in action in the playground above — switch the dropdown and watch the output's behavior change.

5From one neuron to a whole network

One neuron is weak. The magic comes from connecting many of them. Put several neurons side by side → that's a layer. Stack layers front to back → that's a neural network.

A small network. Each circle is a neuron exactly like Section 2. Each line is a weight.
Why "deep learning"? It literally just means a network with many hidden layers — it's "deep" because it's many layers tall. That's the whole origin of the word.

What the hidden layers actually do (the real superpower)

Each layer learns to detect more abstract things, automatically, by combining what the previous layer found. For recognizing a face in a photo:

Layer 1 → detects tiny edges and color blobs.

Layer 2 → combines edges into shapes: curves, corners.

Layer 3 → combines shapes into parts: an eye, a nose, a mouth.

Output → "those parts in this arrangement = a face." ✅

Nobody programmed "what an edge is" or "what an eye is." The network discovered these useful building blocks on its own while learning. That automatic discovery is what makes deep learning so powerful compared to older methods.

6Making a prediction — the "forward pass"

Running data through the network from input to output to get an answer is called the forward pass. (When you chat with an AI, this is what's happening — it's also called inference.) Let's do one by hand with real numbers so it's concrete. Click through the steps.

Tiny example: one neuron deciding "should I go outside?" from two inputs — x₁ = temperature niceness (0.8) and x₂ = rain (0.9). We'll use weights w₁=1.0 (nice weather → yes) and w₂=−2.0 (rain strongly → no), bias b=0.5, sigmoid activation.

7How a network learns

So far the weights were just sitting there. Where do good weights come from? They start as random numbers (the network begins by guessing nonsense), and then a training loop fixes them. Here is the entire learning loop — it's only four steps and it's the heartbeat of all of deep learning:

  • 1. Predict. Run the forward pass with current (initially random) weights. Get an answer.
  • 2. Measure the error. A loss function compares the prediction to the correct answer and produces one number: how wrong we were. Big number = very wrong. The entire goal of training is to make this number small.
  • 3. Assign blame (backpropagation). The error is traced backward through the network to figure out how much each individual weight contributed to the mistake. This gives a direction for each weight: "nudge me up" or "nudge me down."
  • 4. Adjust. Every weight is nudged a tiny step in the direction that reduces the error. How big a step is the learning rate.
Backprop vs. gradient descent — people confuse these:
Backpropagation = the method that figures out which way each weight should move (it computes the blame).
Gradient descent = the method that actually moves the weights downhill toward less error.
Together they are the training engine. Next section makes "downhill" visual.

8Gradient descent — rolling downhill, visually

Picture the "loss" (the wrongness number) as a landscape of hills and valleys. Every possible set of weights is a position on this landscape. High ground = bad weights (lots of error). The deep valley = the best weights (least error). Gradient descent is simply: feel which way is downhill, take a small step that way, repeat — until you reach the bottom.

Each "step" is one weight update from the loop in Section 7. A real network does this for millions of weights at once, across a landscape with millions of dimensions — but the idea is exactly this ball, finding the valley. The learning rate is the step size: too big and the ball leaps over the valley and bounces forever; too small and it crawls and training takes ages.

9Putting it all together

You now have the complete picture. Read this paragraph — every term in it is something you just learned:

A neural network is a stack of layers made of neurons. Each neuron does weight · input + bias, then an activation adds the bend that lets the network learn complex shapes. To get an answer you run a forward pass (also called inference). The weights start random, so the answers start terrible. A loss function measures how wrong they are; backpropagation assigns blame to each weight; and gradient descent nudges every weight a small learning rate-sized step downhill toward less error. Repeat millions of times and the weights settle into values that produce correct answers. Those final settled weights are the learned knowledge — and when a model has "billions of parameters," it has billions of these weights.
