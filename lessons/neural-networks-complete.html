CHAPTER 1

What a neural network is

A neural network is a machine that learns to turn inputs into the right answer — not by you writing rules, but by it adjusting numbers from examples.

Normal program: you write the rule. if temperature > 100: alert(). You decide everything.

Neural network: you show examples, it finds the rule itself. You give it 10,000 photos labelled "cat / not cat," and it works out the pattern — including patterns no human could write down.

Neural network — a big pile of numbers (called weights) plus a simple formula that turns an input into an output. "Learning" = slowly adjusting those numbers until the answers are good.

That's the one sentence the whole lesson builds on. The rest is just: what are those numbers, how does the formula work, and how do they get adjusted?

Our one example (we'll use it the whole way)

To avoid confusion, we use a single example for the entire lesson. Here it is:

Goal: predict whether a plant is healthy (answer near 1 = healthy, near 0 = not).

Inputs (the two numbers describing this plant):

  • sunlight = 0.5
  • water = 1.0

The true answer for this plant: 1 (it really is healthy). The network must learn to output something close to 1.

Where do the numbers come from? The inputs (0.5, 1.0) are measurements of one plant. The true answer (1) is the correct label we already know. Every other number you'll meet (weights, biases) starts as a random guess and gets improved by training. Nothing is magic — every number has an origin.

CHAPTER 3

The neuron: inputs & weights

The whole network is built from one tiny part: the neuron. A neuron takes the inputs, and does the simplest useful thing — it weighs them.

Weight — a number attached to one input that says "how much does this input matter?" Each input gets its own weight. Big weight = this input matters a lot; negative weight = this input pushes the answer down.

Step one of a neuron: multiply each input by its weight, then add the results. Let's do it for one neuron — call it h₁ — with weights 0.4 and −0.2 (random starting values):

sunlight × its weight = 0.5 × 0.4 = 0.20
water × its weight = 1.0 × (−0.2) = −0.20
add them: 0.20 + (−0.20) = 0.00

So far h₁ has computed 0.00. This "multiply each input by its weight and add" is called the weighted sum. That's the heart of every neuron.

Why does ONE neuron have TWO weights? Because it has two inputs, and each input needs its own weight. 5 inputs → 5 weights. The neuron weighs every input it receives.

CHAPTER 4

The bias

After the weighted sum, the neuron adds one more number: the bias.

Bias — a single number that belongs to the neuron (not to any input). It's added after the weighted sum. It shifts the neuron's result up or down — think of it as the neuron's "default lean" before inputs arrive.

h₁'s bias is 0.1. Continue from the 0.00 we had:

weighted sum = 0.00
+ bias = 0.00 + 0.1 = 0.10
Important: the bias lives inside the neuron, not on a wire/connection. So if a neuron has 2 input-weights, it still has just 1 bias. The weights are on the wires; the bias is in the neuron.

CHAPTER 5

The activation function

The last thing a neuron does: pass its result through an activation function.

Activation function — a small function that "bends" the neuron's output. The most common is ReLU: if the number is negative → make it 0, otherwise keep it.

h₁ had 0.10. ReLU of a positive number leaves it alone:

ReLU(0.10) = 0.10 → h₁ is finally 0.10

Why do we even need it?

Without an activation, stacking many neurons would collapse into one big straight-line formula — it could only learn straight-line relationships. Real problems (images, language, our plant) are full of curves. The activation adds the "bend" that lets the network learn complicated shapes. That bend is the whole reason deep networks are powerful.

ActivationWhat it doesUsed for
ReLUnegatives → 0, else keepthe default, inside most networks
Sigmoidsquish any number into 0…1output as a probability
Softmaxturn a list into probabilities summing to 1picking among many options

CHAPTER 6

Layers & a full network

One neuron is weak. Power comes from connecting many. Put several neurons side by side → a layer. Stack layers → a network.

Our example uses 2 inputs → 3 hidden neurons → 1 output neuron:

Each circle is a neuron (Chapter 3–5). Each line is a weight.
Hidden layer — the neurons in the middle (not input, not output). Deep learning just means a network with many hidden layers — "deep" = many layers tall.

Count the numbers: 2×3 = 6 weights in, 3×1 = 3 weights out (9 weights), plus 1 bias per hidden + 1 for output = 4 biases. 9 + 4 = 13 parameters.

Parameters — all the weights + all the biases together. They're the network's entire learned knowledge. "175 billion parameters" = 175 billion of these numbers.

What do hidden layers do? Each learns to detect more abstract things automatically. For a face photo: layer 1 finds edges → layer 2 combines them into shapes → layer 3 into eyes/noses → output says "face." Nobody programmed "edge" or "eye" — the network discovered them.

CHAPTER 7

The forward pass (with real numbers)

Forward pass — running the inputs through the whole network, left to right, to get the answer. (When you chat with an AI, this is what happens — also called inference.)

Let's run our plant all the way through. Every neuron does Chapter 3–5: weighted sum → +bias → ReLU. Here are all 13 numbers in action:

Hidden layer (inputs: sun=0.5, water=1.0)
h₁: 0.5×0.4 + 1.0×(−0.2) + 0.1 = 0.10 → ReLU → 0.10
h₂: 0.5×0.1 + 1.0×0.7 + (−0.3) = 0.45 → ReLU → 0.45
h₃: 0.5×(−0.5)+ 1.0×0.3 + (−0.2) = −0.15 → ReLU → 0.00 (negative, clipped to 0)

Output neuron (inputs: h₁,h₂,h₃ ; weights 0.6,−0.4,0.5 ; bias 0.05)
0.10×0.6 + 0.45×(−0.4) + 0.00×0.5 + 0.05 = −0.07

Prediction = −0.07

So with its current (random) weights, the network predicts −0.07. The true answer was 1. It's badly wrong — which is expected, because the weights are still random. Fixing that is the rest of the lesson.

CHAPTER 8

The loss: measuring wrongness

To improve, the network first needs to know how wrong it is, as a single number.

Loss (also called error — same thing) — one number saying how wrong the prediction is. Bigger = more wrong. 0 = perfect. A common formula is squared error: (prediction − answer)².
prediction = −0.07, true answer = 1
loss = (−0.07 − 1)² = (−1.07)² = 1.1449

So our loss is 1.14. The entire goal of training is to make this number as small as possible. Everything from here is about shrinking the loss.

Two words, one meaning: "loss" and "error" are the same thing. Don't let the two names confuse you.

CHAPTER 9

What "learning" means

The network has 13 parameters (knobs). Right now they're random, so the loss is high. Learning = turning the knobs to make the loss small.

To turn a knob the right way, we need to answer one question for each knob:

The one question: "If I nudge this weight up a tiny bit, does the loss go up or down — and by how much?"

If nudging a weight up makes the loss go down, we should increase that weight. If it makes the loss go up, we decrease it. Answer this for all 13 knobs, nudge them all the right way, and the loss drops. Do it thousands of times → trained network. That's the whole idea. The next chapters make it precise.

CHAPTER 10

data vs grad (the two slots in every box)

To track all this, think of every number in the network as a little box with two slots:

data = the number itself (its current value).  |  grad = how much this number affects the loss (the answer to "the one question" from Chapter 9).
boxits dataits grad means…
a weighte.g. 0.4how much the loss changes if this weight nudges up
the losse.g. 1.14always 1 (the loss vs itself)
Careful — "data" does not mean "weight." data just means "the number in this box." A weight is one box (its data is the weight value); the loss is another box (its data is the loss value). Every box has a data and a grad.

CHAPTER 11

The gradient (the compass)

Gradient (grad) — a number telling you, for one weight: which way to turn it (the sign) and how strongly it matters (the size). Minus = turn the weight up to reduce loss; plus = turn it down.

Let's see a grad with the simplest possible network so the numbers stay clean: one input x=1, one weight w, prediction = x·w = w, and we want the answer to be 10. Then loss = (w − 10)².

Say w = 4 right now. Let's measure the grad by nudging (this is literally what grad means):

loss at w=4: (4 − 10)² = 36
loss at w=4.01: (4.01 − 10)² = 35.8801
change in loss = 35.8801 − 36 = −0.1199 for a +0.01 nudge
rate = −0.1199 / 0.01 ≈ −12 ← this is the grad

So w.grad = −12. The minus sign says "nudge w up to reduce loss." (There's a shortcut formula for this curve — 2(w−10) = 2(4−10) = −12 — but you just measured it directly, no formula needed.)

CHAPTER 12

Backpropagation (how grad is computed automatically)

For one weight we measured the grad by hand. But a real network has millions of weights — no human can do that. Backpropagation is the machine that computes every weight's grad automatically.

The key: a weight doesn't touch the loss directly — it goes through steps (a trail). Our tiny loss (w−10)² is really two steps:

step 1 (subtract): d = w − 10
step 2 (square): e = d² ← the loss
Each step knows only its own simple "rate":
• subtract step: nudge w up by 1 → d goes up by 1. Local rate = 1.
• square step (e = d², at d=−6): local rate = 2×d = −12.

Why you multiply the rates (the chain rule)

Think of currency: $1 = 80 rupees, 1 rupee = 100 paise, so $1 = 80×100 = 8000 paise. Rates multiply along a chain. Same here — to get how w affects the loss, multiply the rates along the trail.

Walk the trail backward (this is backprop)

e.grad = 1 ← the seed (loss vs itself)
d.grad = (rate d→e) × e.grad = (−12) × 1 = −12
w.grad = (rate w→d) × d.grad = (1) × (−12) = −12

Backprop just rebuilt w.grad = −12 — the same number we measured — by starting with 1 at the loss and multiplying local rates backward. Because each step is simple and local, you can chain a million of them. That's how every weight gets its grad.

CHAPTER 13

Gradient descent & the learning rate

Now that each weight has a grad, we use it to actually update the weight. The rule:

Gradient descent — the update step: new weight = old weight − (learning rate × grad). The minus sign steps the weight against its grad — i.e. downhill, toward less loss.
Learning rate — how big a step we take. A small number like 0.1. Too big → we overshoot and the loss explodes. Too small → training crawls.

Update our w=4 (grad −12), learning rate 0.1:

new w = 4 − (0.1 × −12) = 4 − (−1.2) = 4 + 1.2 = 5.2
new loss = (5.2 − 10)² = 23.04 (was 36 — it dropped!)

One step and the loss fell from 36 → 23. The weight moved toward 10, exactly like the grad's sign said. Picture it as a ball rolling downhill into a valley — the grad is the slope, the learning rate is the step size.

CHAPTER 14

The training loop

Put it all together. Training = repeating four steps over and over:

1. Forward run inputs through → get prediction (Ch 7)

2. Loss measure how wrong (Ch 8)

3. Backward backprop fills every weight's grad (Ch 12)

4. Update nudge every weight downhill (Ch 13)

…then repeat. Each loop, the loss gets a little smaller.

CHAPTER 15

Batches & epochs

Real datasets have millions of examples. Running all of them before each update is slow. So we use batches.

Batch (minibatch) — a small handful of examples (say 32) used for one training step. We compute the loss on the batch, update the weights, then grab the next batch.
Epoch — one full sweep through all the batches (i.e. all the data once). Models train for many epochs.

Why batches? Two reasons: (1) it fits in memory, and (2) the little bit of randomness from using different batches actually helps the network learn better. Most training you'll ever see is: for each epoch → for each batch → forward, loss, backward, update.

CHAPTER 16

Overfitting & generalization

The real goal isn't to do well on the examples we trained on — it's to do well on new, unseen data. That ability is called generalization.

Overfitting — when the network memorizes the training examples instead of learning the general pattern. It aces data it has seen and fails on new data. Like a student who memorized the answer key but didn't learn the subject.

To catch this, we split our data into three piles:

splitused for
training setthe network learns from this
validation setwe check on this during training to tune settings & spot overfitting
test setthe final, honest grade — touched only once, at the very end

The tell-tale sign of overfitting: training loss keeps dropping, but validation loss starts rising. That gap = memorizing, not learning.

CHAPTER 17

Parameters vs hyperparameters

Two similar words, very different meanings:

Parameters — the numbers the network learns by itself (the weights & biases). You don't set these; training does.
Hyperparameters — the settings you choose before training: the learning rate, the number of layers, the number of neurons per layer, the batch size, how many epochs. Tuning these well is a big part of the job.
who sets itexamples

CHAPTER 18

The whole picture (one paragraph)

A neural network is built from neurons, grouped into layers. Each neuron does weighted sum + bias → activation. Running inputs through all the layers is the forward pass, giving a prediction. A loss measures how wrong that prediction is. The weights start random, so the loss is high. Backpropagation walks backward from the loss, multiplying local rates (the chain rule), to compute each weight's gradient — which way and how much it affects the loss. Gradient descent then nudges every weight a small learning-rate-sized step downhill. Repeat this loop over many batches and epochs, and the loss falls — the network learns. We watch a validation set to avoid overfitting, so it generalizes to new data. The learned weights and biases are the parameters; the settings we chose are hyperparameters.

If you can read that paragraph and understand every word, you understand neural networks. That's the entire foundation — the same machinery scales up to GPT.
