PART 1

The big map: AI → ML → DL → Neural Networks

These four words are nested inside each other, biggest to smallest:

WordMeaning (as you said it)
AIMaking a machine do a task intelligently — making a machine smarter. The biggest umbrella.
Machine Learning (ML)A way to make a machine smarter by learning patterns from data instead of hand-written rules.
Deep Learning (DL)ML done with neural networks that have many layers ("deep" = many layers).
Neural Network (NN)The machine itself: a pile of numbers (weights) + a formula that turns inputs into an output.

So: AI ⊃ ML ⊃ Deep Learning ⊃ (uses) Neural Networks.

Normal program vs neural network — In a normal program (encoder/rules), you declare the rules manually and it calculates. In a neural network, you give it data (this is right, this is not right) and the network figures out the rules itself. That flip is the whole point.
Deep learning, precisely: a stack of neurons = a layer; a stack of layers = "deep"; when the whole thing runs and trains many times, that's deep learning. We are working inside this box.
PART 2

Every core concept, explained in detail

Each concept: what it is, the small details, and the exact clarification we made together.

✓The box (Value) — the two slots

Box (Value) — every single number in the network lives in a "box." A box holds two numbers: data and grad.

Picture a sticky note on every number, with two lines on it:

┌─────────────┐
│ data = 0.4  │  ← the value
│ grad = -12  │  ← its effect on the loss
└─────────────┘
  • Forward pass writes the top number (data).
  • Backward pass writes the bottom number (grad).
The key clarification: a box is NOT a neuron. A box is one number. A neuron is made of many boxes (its weights, its bias, its output). Don't confuse the two.

✓data — the value itself

data = the current number inside a box. Just "the value here."
Clarification you asked for: data does not always mean "weight." It's a general word. A weight-box's data is a weight; the loss-box's data is the loss; an input-box's data is the input. data = whatever number this box holds, depending on which box you look at.

✓grad — effect on the loss (sign + number)

grad (gradient) = how much this box affects the final loss. It has two parts: a sign and a number.
part of gradwhat it tells youwhere it's used
sign (+ / −)which way to turn the knob. minus → turn weight UP (to reduce loss); plus → turn weight DOWN.gives direction
number (size)how strongly this weight affects the loss = how big a step it takes.in gradient descent: new = old − lr × grad
What the NUMBER does (your final doubt): the size of the grad decides how big the weight's update is. Big grad → big move; small grad → tiny move. That's why training slows down near the answer — as loss flattens, grad shrinks toward 0, so steps get tiny and the weight settles.

Default: grad starts at 0 for every box, and stays 0 until backward() runs.

✓weight — the line, and a box

Weight — a learned number on one input, saying "how much does this input matter?" In the drawing, the weight IS the line connecting two circles. That line is also a box (data = the weight value, grad = its effect on loss).
  • Each input to a neuron has its own weight. 2 inputs → 2 weights into that neuron.
  • Negative weight → that input pushes the answer down.
  • Default: weights start as random numbers, then training improves them.

✓bias — one per neuron, inside it, and it LEARNS

Bias — a single number that belongs to a neuron (not to any input, not on a wire). It's added after the weighted sum, inside the neuron. It shifts the neuron's result up or down.
  • One bias per neuron. A layer of 3 neurons → 3 biases. Count the circles (except inputs) → that's the bias count.
  • The bias DOES change during training — it's a parameter exactly like a weight. It has a grad, and gradient descent updates it: new bias = old bias − lr × grad.

✓input — fixed, never changed

Input — the numbers describing the one example you feed in (e.g. sunlight = 0.5, water = 1.0). They're the data / the question.
Clarification you asked for: inputs do affect the loss, but we never change them — they're given. The only boxes we ever update are weights and biases. Inputs and the true answer stay fixed.

✓the neuron — the building block

Neuron — a small machine that does three steps: (1) multiply each input by its weight and add them (the weighted sum), (2) add its bias, (3) apply an activation.
neuron output = activation( in₁×w₁ + in₂×w₂ + … + bias )
                            └──── weighted sum ────┘   └┬─┘
                                                      bias

What boxes a single neuron involves: its input boxes (from before), its weight boxes (the lines in), its one bias box, and the output box it produces.

✓activation function — bends the line into a curve

Activation function — a small function applied at the end of the neuron that "bends" the result. Without it, the whole network would collapse into one straight line and could only learn straight-line relationships. The bend is what lets it learn complex patterns.
namewhat it doesused for
ReLUif negative → 0, else keep itthe default, inside most networks
Sigmoidsquish any number into 0…1output as a probability
Softmaxturn a list into probabilities that sum to 1picking among many options

✓layer — a column of neurons; how they connect

Layer — one vertical column of neurons. Data flows left → right, one layer at a time. Input layer (first column) → hidden layers (middle) → output layer (last).

Two rules about how neurons connect (your doubt):

  • Neurons in the SAME layer do NOT connect to each other. Each independently takes the same inputs and computes its own output.
  • Every neuron's output feeds EVERY neuron in the next layer (called "fully connected"). So the outputs of layer 1 become the inputs of layer 2, and so on.
LAYER 1            LAYER 2
 n₁ ──┬──────────▶ m₁
      ├──────────▶ m₂     (each m gets BOTH n₁ and n₂;
 n₂ ──┴──────────▶ m₃      n₁ and n₂ never touch each other)
Deep learning = a network with many hidden layers ("deep" = many columns, left to right). More columns = deeper; more neurons in a column = wider.

✓the output neuron — just another neuron

Your doubt: "how is the final output calculated — is it summing all the neurons?" No. The output is just another neuron using the same formula: it takes the previous layer's outputs as its inputs, multiplies each by a weight, adds them, adds its bias, (optionally activation). It's not special — it's the last neuron.

✓forward pass (inference)

Forward pass (also called inference) — running the inputs left → right through every neuron of every layer to get the final output. When you chat with an AI, this is what happens.

✓loss / error — how wrong we are

Loss (same word as error) — one number saying how wrong the prediction is. Bigger = worse. 0 = perfect. A common formula is squared error: (prediction − answer)².
Two clarifications: (1) "loss" and "error" are the same thing. (2) There is only ONE loss box for the entire network — at the very end. Not one per neuron.

✓The TWO different formulas (loss vs grad)

This confused you a lot — so, clearly, there are two separate formulas doing two separate jobs:

Loss formulaGrad formula
tells youhow wrong you arehow fast wrongness changes if you nudge the weight
exampleloss = (w − 10)²grad = 2(w − 10)
think of it asthe height of the groundthe steepness of the slope
rate = grad (SAME thing!): the number you called "the rate" IS the grad. 2(w−10) is the grad, and "rate" is just another name for it. One number, two names — not two separate calculations. It's the rate of change of the loss as you change the weight, i.e. the slope of the loss.

✓backpropagation — computing all the grads

Backpropagation — the process that computes the grad of every box automatically. It starts at the loss, walks backward to the first layer, and fills in each box's grad.

How it works (three ideas):

  • Seed: start at the loss with loss.grad = 1 (the loss's effect on itself is always 1).
  • Local rate: each little step knows only its own simple rate (e.g. "add" step's rate = 1; "square" step's rate = 2×its input).
  • Chain rule: multiply the local rates backward along the trail. (Like currency: $1 = 80 rupees, 1 rupee = 100 paise, so $1 = 80×100 = 8000 paise — rates multiply along a chain.)
The Karpathy word "derivative": "derivative of g with respect to a" is just fancy math for "a's grad" = "how a affects g." derivative = grad = rate. Same thing, three names. Don't let the word scare you.

✓gradient descent — using the grads to update

Gradient descent — the process that updates the weights and biases using their grads. The formula: new = old − (learning rate × grad). The minus sign steps each knob against its grad, i.e. downhill toward less loss.
backprop vs gradient descent (you had this right!): backprop = computes the grads (loss → back to first layer). gradient descent = the next, separate step that moves the weights and biases using those grads. grad = a number · backprop = how we GET it · gradient descent = how we USE it.

✓learning rate — YOU choose it

Learning rate (lr) — how big a step gradient descent takes. A small number like 0.1 or 0.01. You choose it — the network never learns it.

Where the value comes from: trial and tuning. Pick a common value, run training, watch the loss:

loss drops smoothly     → good lr ✅ keep it
loss explodes / bounces → TOO BIG,  make it smaller (÷10)
loss crawls very slowly → TOO SMALL, make it bigger  (×10)

✓the training loop — the whole thing

1. Forward — run inputs through → get prediction

2. Loss — measure how wrong (loss box)

3. Backprop — fill every box's grad (loss → back)

4. Gradient descent — update every weight & bias: new = old − lr × grad

…then repeat, millions of times. Each loop the loss gets a little smaller.

✓parameters vs hyperparameters

who sets itexamples
parameterthe machine learns themweights, biases
hyperparameteryou choose them, before traininglearning rate, number of layers, neurons per layer, batch size

✓bits, bytes & "billions of parameters"

Parameters = all weights + all biases together. "175 billion parameters" = 175 billion of these numbers.
  • bit = one 0/1. byte = 8 bits. Each weight is a number stored in some bytes.
  • float32 = 4 bytes/weight · float16 = 2 bytes · int8 = 1 byte.
  • Memory = (number of weights) × (bytes per weight). The "≈2 bytes" rule is just 16-bit storage.
  • 7B model × 2 bytes = 14 GB · 70B × 2 = 140 GB. This decides what hardware can run it.
  • Quantization = fewer bytes per weight (e.g. int8) → smaller model, fits smaller machines.
  • Your Mac (16 GB) fits ~7B in 16-bit; bigger needs quantization.

One weight = one connection (one line). A model is the same training loop run over millions/billions of these at once — backprop fills every grad in one backward pass.

PART 3

The two worked examples (real numbers)

Example A — the plant network (forward pass, 13 parameters)

2 inputs → 3 hidden → 1 output. Inputs: sunlight=0.5, water=1.0. Activation = ReLU. True answer = 1.

h₁: 0.5×0.4 + 1.0×(−0.2) + 0.1 = 0.10 → ReLU → 0.10
h₂: 0.5×0.1 + 1.0×0.7 + (−0.3) = 0.45 → ReLU → 0.45
h₃: 0.5×(−0.5)+ 1.0×0.3 + (−0.2) = −0.15 → ReLU → 0.00 (negative → clipped)

out: 0.10×0.6 + 0.45×(−0.4) + 0.00×0.5 + 0.05 = −0.07 (prediction)

loss = (−0.07 − 1)² = 1.1449

13 parameters = 9 weights (6 in, 3 out) + 4 biases (3 hidden, 1 output). Run it: train-your-own-gpt/explore_05_network_forward.py.
